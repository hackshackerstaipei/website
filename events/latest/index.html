第 19 場小聚

疫情蔓延新聞室

Hacks/Hackers Taiepi ( http://hackshackers.taipei/ )是全球非營利新聞社群Hacks/Hacker的台北分支，2016年9月由一群關心新聞x科技發展的朋友組成，透過不定期的工作坊、黑客松、案例分享會，鼓勵記者編輯（the “hacks”）和程式設計師（the “hackers”）相互學習、合作，一起想像新聞的未來。

2020年7月12日（日）Hacks/Hackers Taipei的第19次小聚「疫情蔓延新聞室」，特別邀請來自《鏡傳媒Readr》、《天下雜誌》與《《聯合報》的社群之友，與大家分享如何處理和報導新冠疫情，以及專題製作過程中的觀察與想法。

除了疫情數據的討論，這次小聚的bonus將是《報導者》多媒體實驗室的朋友來分享他們製作的open source工具「新聞敘事網頁元件」，可以幫助大家未來在製作過程中避免「重複造輪子」。

由於考量疫情，這次小聚座位有限，我們不開放候補、也不接受現場報名。也請成功報名的朋友，在活動當天「全程配戴口罩」。

若您在過去14天有任何身體不適、曾出國、或曾接觸居家隔離與居家檢疫者，建議您留在家中觀看小聚直播。

分享內容介紹：

主題一：當疫情在編輯部蔓延

  • *《READr》記者兼專題製作人李又如 *
    疫情之下，假訊息如何全球流竄？《READr》如何分析國際事實查核組織超過5000篇事實查核報告，探索假訊息的樣貌，並透過 Twitter 資料挖掘它們的影響力。

  • *《天下雜誌》資料新聞記者陳貞樺 Mimi Chen / 前端工程師高德理 Daniel Kao *
    分享《天下雜誌》數位團隊如何在疫情時間，與台大地理系團隊合作，透過雙北人流數據模擬，模擬什麼樣的封城模式最能阻斷疫情。

  • *《聯合報》資深數位內容製作蔡佩蓉 / 前端工程師楊若榆 *
    如何從「讀者服務」角度出發，串接開放資料，將數據視覺化，即時更新，讓疫情資訊清楚透明。並隨疫情做滾動式修正，讓報導更貼近讀者需求。

主題二：小聚 bonus!

  • 《報導者》前端工程師方泰鈞 / 前端工程師余崇任 / 工程師李法賢 / 資料記者 柯皓翔
    過去5年《報導者》多媒體實驗常依新聞特性量身打造專題，但高成本作品常在新聞過去後乏人問津。為避免「重複打造輪子」，團隊今年開發一系列開源、模組化、可重複使用的「新聞敘事網頁元件」，協助說好疫情故事外，更將此服務公開給所有人使用。
